# AI-Assisted Workflow

This project was developed with AI support during planning, implementation, debugging, refactoring, and test preparation.

The AI support was used as a coding assistant, not as a replacement for understanding the solution.  
All generated ideas, code suggestions, and refactoring proposals were reviewed, adjusted, and tested before being accepted into the project.

## Tools Used

- ChatGPT
- GitHub Copilot

## How AI Was Used

AI was used mainly for:

- understanding the task requirements
- planning the project structure
- discussing backend and frontend architecture
- reviewing implementation ideas
- debugging TypeScript and React issues
- improving component structure
- refining API design
- preparing automated tests
- improving commit messages
- improving README and documentation

## Typical Use Cases

### 1. Task analysis and planning

AI was used to break the assignment into smaller steps and suggest a simple implementation plan.

Examples:

- how to structure the backend and frontend
- how to interpret the ASCII map format
- how to represent tiles and cabanas in the API
- how to keep the solution simple and aligned with the task requirements

### 2. Implementation support

AI helped with:

- TypeScript typing
- React component structure
- utility function extraction
- API integration flow
- booking form logic
- backend validation logic

The code was often rewritten or adjusted manually after discussion and author thinking process.

### 3. Debugging and refactoring

AI was used to investigate:

- frontend rendering issues
- selected cabana state issues
- backend route problems
- TypeScript mismatches
- project structure cleanup

AI also helped suggest refactors such as:

- splitting larger UI logic into smaller components
- moving helper logic into utility files
- simplifying project structure when some abstractions were unnecessary

### 4. Test preparation

AI was used to discuss:

- which tests were the most important for this task
- how to separate backend logic tests from API tests
- how to test frontend booking flow
- how to keep the test scope aligned with the assignment

The final tests were reviewed and adjusted to match the actual project behavior.

### 5. Documentation

AI helped prepare:

- README structure
- test command descriptions
- API endpoint descriptions
- AI workflow documentation

## Prompting Approach

The workflow was iterative and based on many small prompts instead of one large prompt.

Typical prompt types included:

- explaining a bug or error message
- asking for code review
- asking whether a refactor was worth doing
- asking how to simplify a solution
- asking how to structure tests
- asking for better naming or commit messages
- asking for help writing documentation in clear English

## Approximate Number of Steps

The AI-supported workflow took place over many short iterations.  
It was not a single prompt-based generation.

Approximate process:

- many planning and discussion steps
- many implementation and debugging steps
- multiple refactoring steps
- multiple documentation improvements

In total, the solution evolved through dozens of small AI-assisted interactions.

## Verification Process

All important suggestions were verified by:

- reading and understanding the proposed code
- adjusting code manually where needed
- running backend tests
- running frontend tests
- checking UI behavior in the browser
- confirming that the solution still matched the assignment requirements

AI suggestions were not accepted automatically.

## What Was Kept Simple on Purpose

AI was also used to help avoid over-engineering.

Examples:

- no database was added
- no authentication was added
- cabana booking state is stored in memory
- the UI was kept focused on the required flow
- the architecture was kept simple and readable for a code test

## Final Note

AI was used as a thinking and support tool during development, but the final solution was reviewed, tested, and shaped manually step by step.
